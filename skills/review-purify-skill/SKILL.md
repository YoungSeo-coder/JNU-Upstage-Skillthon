# 알바 리뷰 순화 스킬 (review_purify_skill)

알바 후기 주관식 텍스트를 입력받아 법적으로 안전한 표현으로 순화하고, 변경 이유와 리스크 등급을 JSON으로 반환한다.

## 실행 방법

`./scripts/purify.py`를 실행한다. `UPSTAGE_API_KEY` 환경변수가 필요하다.

```bash
python ./scripts/purify.py "사장 성질 더럽고 월급도 맨날 늦게 줌"
```

또는 사용자가 텍스트를 제공하면 직접 스크립트를 호출한다.

## 입력

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `review_text` | string | 필수 | 사용자 주관식 후기 원문 |
| `business_id` | string | 선택 | 연동 사업장 ID |

## 출력 형식

```json
{
  "purified_text": "순화된 후기 텍스트",
  "risk_level": "HIGH | MEDIUM | LOW | SAFE",
  "extracted_tags": ["급여지연", "소통어려움"],
  "changes": [
    {
      "original": "원문 표현",
      "replaced": "순화된 표현",
      "reason": "순화 이유 (관련 법 조항 포함)"
    }
  ]
}
```

## 순화 기준

핵심 원칙: **사람의 성격·인성·태도·말투를 직접 평가하지 않는다. 근무자가 경험한 상황과 업무 환경 중심으로 서술한다.**

탐지 후 순화할 표현 유형:

1. **명예훼손 소지** — 특정인 지칭 모욕, 검증 불가 사실 단정 (형법 제307조)
2. **욕설·혐오 표현** — 비속어, 인격 비하 문장 (정보통신망법 제44조의7)
3. **법적 단정 표현** — '위법이다', '처벌받아야 한다' 등 법적 판단 단정
4. **허위 가능성 표현** — 확인되지 않은 사실을 단정적으로 서술

원래 의미(사실 내용)는 손상되지 않도록 유지한다.

## 리스크 등급 기준

| 등급 | 기준 |
|---|---|
| HIGH | 명예훼손 소송 위험이 높은 표현 포함 |
| MEDIUM | 욕설·감정 표현이 포함되어 삭제 위험 있음 |
| LOW | 경미한 주관적 표현, 권고 수준의 순화 |
| SAFE | 법적 문제 없음, 순화 불필요 |

## 순화 예시

| 입력 (원문) | 출력 (순화본) |
|---|---|
| "사장님이 미쳐서 소리 지름, 절대 가지 마세요 개쓰레기임" | "고성이 오가는 상황이 발생한 경험이 있으며, 근무 중 심리적 부담을 느낀 적 있음" |
| "사장이 수시로 CCTV 보면서 카톡으로 쌍욕함" | "근무 중 수시로 CCTV 확인 및 메신저를 통한 부적절한 언어 사용이 있었음" |
| "알바생을 노예로 암. 쉬는 시간 1분도 안 줌" | "업무 강도가 높은 편이며, 법정 휴게시간이 보장되지 않은 경험이 있음" |
| "사장 성질 더럽고 월급도 맨날 늦게 줌" | "급여 지급이 반복적으로 지연된 경험이 있음" |

## 환경 설정

`UPSTAGE_API_KEY` 환경변수가 설정되어 있어야 한다. 없으면 즉시 오류를 반환한다.
