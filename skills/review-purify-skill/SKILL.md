# 알바 후기 순화 스킬 (review-purify-skill)

사용자가 작성한 알바 후기에서 법적 리스크 표현을 탐지하고, 뉘앙스가 다른 3가지 순화 버전을 제안한다. 사용자가 직접 선택하여 제출하는 방식으로 작성자의 선택권을 보장한다.

## 처리 파이프라인

```
사용자 자유 텍스트 입력
    ↓
Solar LLM 분석    (명예훼손·욕설·법적 단정 표현 탐지)
    ↓  risk_assessment
3가지 순화 버전 생성  (건조한 사실형 / 완곡형 / 감정 유지형)
    ↓
결과 JSON 반환 → 사용자 선택 → 최종 제출
```

## 실행 방법

`./scripts/purify.py`를 실행한다. `UPSTAGE_API_KEY` 환경변수가 필요하다.

```bash
# 커맨드라인 인수
python ./scripts/purify.py "사장 성질 더럽고 월급도 맨날 늦게 줌"

# stdin JSON 입력
echo '{"review_text": "쉬는 시간도 없고 계약서도 안 썼어요"}' | python ./scripts/purify.py
```

## 입력

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `review_text` | string | 필수 | 순화할 원본 후기 텍스트 |

## 출력 형식

```json
{
  "original_text": "사장 성질 더럽고 월급도 맨날 늦게 줌",
  "risk_assessment": {

    },
    {
      "option_id": 2,
```

## 리스크 등급

| 등급 | 기준 |
|---|---|
| HIGH | 명예훼손 소송 위험이 높은 표현 포함 |
| MEDIUM | 욕설·감정 표현이 포함되어 삭제 위험 있음 |
| LOW | 경미한 주관적 표현, 권고 수준의 순화 |
| SAFE | 법적 문제 없음, 순화 불필요 |

## 순화 원칙

- **사람 평가 금지**: 고용주·관리자의 성격·인성·태도를 직접 평가하지 않는다
- **경험 중심 서술**: 근무자가 실제로 겪은 상황과 업무 환경 중심으로 표현한다
- **사실 내용 보존**: 순화 과정에서 원래 사실 내용은 손상되지 않도록 유지한다

## 사용자 선택 및 동의 프로세스

AI가 일방적으로 리뷰를 변환하지 않는다. 3가지 순화 버전을 제안하고 사용자가 직접 선택한다.

1. 사용자가 자유롭게 후기 텍스트 입력
2. AI가 리스크 분석 + 3가지 순화 버전 반환
3. 사용자가 원문↔순화본 비교 후 선호하는 버전 선택
4. 선택한 버전으로 최종 제출

작성자의 의도를 존중하고 최종 선택권을 보장하는 것이 핵심이다.

## 환경 설정
